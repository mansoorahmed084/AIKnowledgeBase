GenAI: Foundational Concepts

AI (Artificial Intelligence)

Building machines that perform tasks requiring human intelligence

ML (Machine Learning)

Subset of AI where machines learn from data

Generative AI

Application of AI that creates new content
Data Quality
Accuracy
Inaccurate data leads to incorrect patterns and faulty predictions
Completeness
Adequate dataset size and representation for accurate predictions
Representative
Inclusive data prevents skewed samples and biased outcomes
Consistency
Consistent formats and labeling enable effective learning
Relevance
Data must be relevant to the AI's intended task
Data Accessibility
Availability
Data must be available, not locked behind paywalls or privacy restrictions
Cost
Gathering and cleaning data can be expensive, a barrier for smaller organizations
Format
Data needs to be in a format the AI model can understand and process
Data Types
Structured Data
Organized data in tables (databases, spreadsheets) - easy to search and analyze
Unstructured Data
Messy, complex data (text, images, audio, video) requiring sophisticated analysis

Supervised Learning

Trains on labeled data to learn input-output relationships. Example: Predicting housing prices from features
Google Cloud: Predictive maintenance with Vertex AI

Unsupervised Learning

Finds patterns in unlabeled data. Example: Discovering customer segments from purchase history
Google Cloud: Anomaly detection with BigQuery ML

Reinforcement Learning

Learns through interaction and feedback, maximizing rewards. Example: Robot navigation or game playing
Google Cloud: Product recommendations with Vertex AI
ML Lifecycle: Turning Data into Learning

1. Gather

Data ingestion from various sources. Tools: Pub/Sub (streaming), Cloud Storage (unstructured), Cloud SQL/Spanner (structured)

2. Prepare

Clean and transform raw data. Tools: BigQuery (analysis), Data Catalog (governance & discovery)

3. Train

Create ML model using data. Tool: Vertex AI (managed training environment with prebuilt containers and compute resources)

4. Deploy

Make trained model available for use. Tool: Vertex AI (scaling and prediction generation)

5. Manage

Maintain models over time: versioning, performance tracking, drift monitoring, feature store, model garden, pipelines

Key Takeaway

Data is the foundation of AI systems. Understanding data quality (accuracy, completeness, representativeness, consistency, relevance) and accessibility (availability, cost, format) is crucial. The ML lifecycle encompasses gathering, preparing, training, deploying, and managing models, with Google Cloud providing comprehensive tools for each stage.
