Factors for Choosing a Model

Modality

Type of data (text, images, video, audio) - single or multimodal

Context Window

Amount of information model can consider - balance with cost

Security

Encryption, access controls, compliance with standards

Availability

Uptime guarantees, redundancy, disaster recovery

Cost

Pricing model based on usage/compute - match model to task

Performance

Accuracy, speed, efficiency - evaluate benchmarks

Fine-tuning

Customization capabilities for specific tasks/domains

Integration

Well-documented APIs and SDKs for easy integration

Gemini

Multimodal model - understands text, images, audio, video. Supports complex multimodal understanding, conversational AI, content creation

Gemma

Lightweight, open models built on Gemini research. User-friendly for local deployments and specialized applications

Imagen

Text-to-image diffusion model. Generates high-quality images from text descriptions for design and ecommerce

Veo

Video generation model. Creates videos from text or images for film production, advertising, online content

Data Dependency

Requires large datasets - biases and incompleteness affect outputs

Knowledge Cut-off

Models don't know about events after training date - can provide outdated information

Bias

Can amplify existing biases from training data - statistical learning nature

Fairness

Complex to define - evaluations focus on specific categories, may miss other forms

Hallucinations

Produces outputs not based on real information - can't verify against external sources

Edge Cases

Rare scenarios expose weaknesses - errors and misinterpretations

Grounding

Connects AI output to verifiable sources - reduces hallucinations, anchors responses, builds trust with citations

RAG (Retrieval-Augmented Generation)

Uses semantic search to find relevant info from knowledge base, augments prompt, then generates response

Prompt Engineering

Craft precise prompts to guide model outputs - rapid approach but limited by model's existing knowledge

Fine-tuning

Further trains pre-trained model on task-specific dataset - adjusts parameters for specialization

Humans in the Loop (HITL)

Content Moderation

Human oversight for accurate, contextually appropriate content filtering

Sensitive Applications

Healthcare/finance oversight for critical decisions and risk mitigation

High Risk Decisions

Human review for medical diagnoses, criminal justice assessments

Pre/Post Generation Review

Review before deployment and continuous feedback after deployment

RAG vs Fine-tuning vs Grounding Comparison

Feature RAG Fine-tuning Grounding
Definition Augments LLMs by retrieving relevant information from external knowledge bases and adding it to the prompt Further trains a pre-trained model on a new dataset to adapt it to a specific task or domain Connects an AI model's output to verifiable sources of information
Process Retrieve relevant information → Add it to the prompt → Generate a response Select a pre-trained model → Gather data → Train the model → Evaluate and refine Provide access to data sources → Use RAG or fine-tuning to connect the output
Data Sources External knowledge bases (databases, documents, internet) Task- or domain-specific datasets External knowledge bases or specific datasets
Relationship to Grounding A specific technique for achieving grounding Improves a model's ability to be grounded in specific domains The overarching goal, achieved through techniques like RAG and fine-tuning

Key Takeaway

Deep learning powers foundation models (LLMs, diffusion models) that enable generative AI. When choosing models, consider modality, context, security, cost, and performance. Foundation models have limitations (hallucinations, bias, knowledge cut-offs) that can be addressed through grounding techniques like RAG, prompt engineering, fine-tuning, and human-in-the-loop oversight. Grounding is the overarching goal of connecting AI outputs to verifiable sources.
